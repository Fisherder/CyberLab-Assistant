# 答案草稿

这个文件可以用来临时整理思路。最终答案仍需要填写到网页右侧“提交”区域。

## 根因说明

登录接口把用户提交的字段用于身份判断时，没有把输入数据和查询逻辑可靠分离，导致特殊输入可能改变原本的判断条件。

请用自己的话说明：

- 哪个输入字段会影响认证判断；
- 认证判断为什么会从“账号密码错误”变成“登录成功”；
- 这类问题属于哪种输入信任边界错误。

## 验证过程

请记录：

- 目标地址；
- 正常错误账号请求的状态码和响应体；
- 异常输入请求的状态码和响应体；
- 你如何确认返回的是管理员或高权限身份。

## 修复建议

请说明：

- 查询逻辑应如何处理用户输入；
- 密码校验和身份判断应放在什么边界内完成；
- 修复后应该如何回归测试正常失败、正常成功和恶意输入三类情况。
