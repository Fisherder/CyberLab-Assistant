# ADR 0006：原始终端保留与隐私

## 状态

已采纳。

## 背景

规格要求原始终端字节与语义事件分离，并禁止动态秘密进入普通日志、Trace、Prompt 和公开录制。

## 决策

P1 代码把语义事件存入 `events`，原始终端分片只在 `transcript_segments` 中建立索引，实际对象由 API 端使用 AES-GCM 加密后写入对象存储。存储后端可配置为开发用 `local` 或兼容 MinIO/S3 的 `s3`。Gateway 异步上传分片到内部 API；API 加密分片、只在数据库保存对象元数据，并提供内部恢复校验和保留清理接口。恢复校验只检查对象哈希与可解密性，不返回明文终端内容。Gateway/sessiond 代码不记录终端 payload。

## 影响

本地文件后端用于开发和测试。S3/MinIO 路径已用 fake S3 client 覆盖写入、读取、删除和 bucket/prefix 绑定，但生产仍需要 live MinIO/S3 smoke、桶生命周期策略、理由审计访问控制和对象存储恢复演练。

## 回滚

若某课程禁用原始终端保留，保留语义事件和成绩证据即可。
